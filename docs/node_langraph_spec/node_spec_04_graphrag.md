## 1. 입출력 계약

### 1-1. 입력

| 키 | 타입 | 채우는 주체 | 없으면 / None이면 |
|---|---|---|---|
| `group["pattern"]` | `str` | ② grouper (← ① CNN 라벨) | **필수**. 3종 밖이면 진입 경로가 달라짐(§3) |
| `group["observation"]` | `Observation \| None` | ③ vlm_describe | **없어도 동작**. 있으면 후보를 훨씬 정확히 좁힘 |
| `KG_LIVE` (env) | `1/true/yes` | `.env` | 미설정(기본) = 파일 조회 |
| `KG_HYPOTHESES_PATH` (env) | 경로 | `.env` | 파일 경로에서 **필수** — 없으면 기동 실패 |
| `KG_SEMANTIC` / `KG_SEMANTIC_MIN_SCORE` | env | `.env` | 미설정 = 자연어 진입 켜짐 / 하한 0.4 |

**`observation`에서 실제로 꺼내 쓰는 것** — 5가지이고 쓰임새가 서로 완전히 다르다.

| 관측 필드 | 무슨 뜻인가 | 여기서 뭘 하는가 |
|---|---|---|
| `location_text` + `morphology_text` (없으면 `description`) | VLM이 쓴 자연어 설명 | 그래프의 **어디서부터 탐색을 시작할지** 정함 (1순위) |
| `signature` | 형상 코드 (`ring@edge` 형식) | 자연어가 없을 때 **대신 시작점을 정함** (2순위) |
| `angular_coverage`, `clock_positions` | 원 전체를 도는가(`full`) 한쪽만인가(`partial`), 몇 시 방향인가 | 안 맞는 후보를 **목록에서 제거** |
| `density`, `continuity` | 진한가 옅은가 / 이어졌나 끊겼나 | **순위만** 조금 내림 |
| `pattern_candidate` | CNN 라벨 사본 | ⚠️ 안 씀 — 노드는 `group["pattern"]`을 본다 |

> **왜 나눠 취급하나**: `angular_coverage`는 이미지 계산으로 정확히 나오는 값이라 믿을 수 있고, `density`·`continuity`는 VLM이 글로 쓴 걸 해석한 값이라 흔들린다. 믿을 수 있는 신호는 강하게, 흔들리는 신호는 약하게만 반영한다.

**전제조건**
- `pattern`은 3종(`Center`/`Edge-Ring`/`Scratch`) 또는 그 밖(`Unknown` 등)이다. `Normal`은 여기 오기 전에 걸러져야 한다(실제로는 `graph.py::_run_groups`의 방어 가드가 건너뜀).
- 파일 경로는 `hypotheses.json`이 현재 그래프와 정합하다고 **가정만 한다**. 신선도 검사 없음.

**가정이 깨지면**: 3종 밖 패턴은 예외가 아니라 **빈 후보 목록**으로 처리된다(의도된 설계, §4-1). 파일 부재·JSON 손상은 예외가 그대로 올라가 그 그룹 분석이 중단된다.

### 1-2. 출력

| state 키 | 타입 | 비고 |
|---|---|---|
| `graphrag_candidates[group_id]` (배치 그래프) / `candidates` (그룹 서브그래프) | `GraphRAGResult` = `{pattern, candidates: [...]}` | **빈 리스트 가능** → `route_on_candidates`가 ⑦'로 분기 |
| `entry_signatures` | `list[str]` (라이브 + 진입 성공 시만) | 어느 형상으로 들어왔는지 — 감사용 |

**후보 하나에 담기는 것 — ⑤ Hypothesis가 실제로 꺼내 쓰는 필드**

| 필드 | 뜻 | 어디에 쓰이나 |
|---|---|---|
| `cause` | 원인 이름 | 가설의 본체. ⑤⑥⑦ 끝까지 따라감 |
| `step` | 어느 공정에서 생기는 원인인가 (`ETCH` 등) | ⑤가 **어느 장비를 볼지** 정하는 열쇠 |
| `mapped_process` | `step`이 비었을 때의 대체 공정 | ⑤ 폴백 전용. `step`이 있으면 **절대 안 씀**(D14) |
| `tier` | 확인 난이도 3단계 (아래) | ⑤가 어떤 도구를 부를지, ⑥이 어떻게 판정할지 |
| `evidence` + `evidence_label` | 무엇을 확인해야 하나 (`rf_power` / `Parameter`) | ⑤의 조회 키 |
| `fab_table` | 어느 테이블을 볼지 | ⑤ |
| `direction` | 값이 올라가야 하나 내려가야 하나 | ⑤가 실제 추세와 대조 |
| `rank` | 조사 순서 (문헌 근거 많은 순) | ⑤의 최종 타이브레이커 |
| `sentence` | 사람이 읽는 가설 문장 | ⑥ faithfulness 대조 · ⑦ 카드에 그대로 실림 |
| `citations` | 어느 문헌에서 왔는지 | ⑦ 근거 모달 (API §2.5) |
| `morphology` | 이 후보가 전제하는 결함 모양 | ④ 내부 필터링용 |
| `matched_cause` | 정답지 대조용 번역 이름 | **평가 전용** — 화면 표시·판정에 쓰면 안 됨 |

**`tier`(확인 난이도) 3단계** — 뒤 단계의 동작을 크게 가른다.

| 값 | 뜻 | 뒤에서 어떻게 되나 |
|---|---|---|
| `자동` | 센서값으로 기계가 판정 가능 | ⑤가 telemetry 조회 → 정상범위 이탈 계산 |
| `반자동` | 정비 이력·레시피 등 사람이 읽어야 함 | ⑤는 자료만 수집, 판정은 미룸 |
| `근거없음` | 문헌엔 있지만 우리 공장 데이터로 확인 불가 | ⑥이 "판단 보류"(P5) |

**하위 노드가 믿어도 되는 불변식** — 깨지지 않게 지키는 약속이다.

1. 후보 목록은 **항상 리스트**다. 조회 실패든 미매핑이든 `None`이 아니라 빈 리스트.
2. `tier`는 반드시 `자동`/`반자동`/`근거없음` 중 하나. 다른 값은 안 나온다.
3. `tier == "근거없음"` ⇒ `evidence`는 반드시 비어 있다.
4. `citations`는 항상 리스트. 인용이 없으면 `[]`이지 `None`이 아니다(API §2.5).
5. **목록 순서 = 조사 우선순위.** 앞에 있을수록 먼저 확인할 후보.
6. 관측한 모양과 정면으로 어긋나는 후보는 **목록에 아예 없다**(순위만 낮추는 게 아니라 제거).
7. ⚠️ **`rank` 값과 목록 순서가 항상 일치하지는 않는다** — §8-1.

### 1-3. 새로 도입 / 변경한 필드

`state.GraphRAGCandidate` 타입 정의에 **없는데** 실행 중에 붙는 키가 3개 있다.

| 키 | 붙이는 곳 | 용도 |
|---|---|---|
| `morphology_score` | `morphology_rank` | 감점 합(≤0). 정렬 키 |
| `entry_signature` | 라이브 자연어 진입 | 어느 형상으로 들어왔나 |

→ `state.py`에 정식으로 올릴지 결정 필요(§8-2).

## 2. 실패·경계 케이스 계약

| 이런 상황이면 | 이 노드는 | 하위가 보게 되는 것 |
|---|---|---|
| 모르는 패턴 + 형상 정보도 없음 | 정상 종료, 빈 목록 | ⑦' `status="unmapped"` — "판독까지만 지원" 안내 (UC-3) |
| 모르는 패턴인데 **형상 정보는 있음** (라이브) | **후보를 낸다** — 패턴을 안 거치고 모양으로 진입 | 미지 패턴도 RCA 가능 |
| 자연어가 어떤 형상과도 안 닮음 (하한 미달) | 매칭 0건 처리 | 억지 후보 없음 — 환각 억제 |
| 임베딩 인덱스 없음 / API 키 없음 | 경고만 찍고 계속 | 자연어 진입만 꺼지고 나머지는 정상 |
| `hypotheses.json` 없음 / 손상 | **예외 전파** | 그 그룹 분석 중단 (폴백 없음) |
| Neo4j 연결 실패 (`KG_LIVE=1`) | **예외 전파** | 동일 |
| `hypotheses.json`이 그래프보다 낡음 | ⚠️ **못 알아챈다** | 조용히 옛 후보를 내보냄 (§8-3) |

- **폴백을 일부러 안 만든 이유**: ① CNN·③ VLM은 실패해도 대충 값을 채워 파이프라인을 잇는다. ④는 그러지 않는다. **근거 없는 원인 후보를 지어내는 게 이 시스템에서 가장 위험한 실패**이기 때문이다. 없으면 없다고 하고 멈춘다.
- **재시도·타임아웃**: 없음. 파일 1회 읽기 / Cypher 수 회.

## 3. 내부 플로우

```
관측을 받음
   ↓
【1】 그래프의 어디서부터 탐색을 시작할지 정한다   ← 결과를 좌우하는 핵심
   ↓
【2】 거기서부터 정해진 규칙대로 그래프를 훑어 원인들을 모은다
   ↓
【3】 중복 정리
   ↓
【4】 관측한 모양과 안 맞는 후보를 걸러내고 순서를 조정한다
   ↓
후보 목록 완성
```

### 【1】 시작점 찾기 — 우선순위 3단

지식그래프는 넓다. 어디서부터 훑느냐가 결과를 가른다.

**1순위 · 자연어로 찾기 (설계상 메인)**
VLM이 쓴 설명("가장자리를 따라 끊김 없이 도는 진한 띠")을 숫자 벡터로 바꿔, 그래프에 저장된 형상 설명들과 얼마나 닮았는지 비교해 가장 비슷한 것을 고른다.
- CNN이 패턴을 알아냈으면 **그 패턴에 속한 형상들 안에서만** 찾는다 — 패턴은 검색 "범위"를 좁히는 역할.
- 패턴을 모르면 전체에서 찾는다 → **모르는 패턴도 분석된다.**
- **닮은 정도 0.4 미만은 아예 안 쓴다**(§4-2).

**2순위 · 형상 코드로 찾기 (현재 실질 주력)**
자연어가 없으면 이미지 계산이 낸 `ring@edge` 같은 코드로 정확히 일치하는 형상을 찾는다. 지금은 VLM 자연어가 미연동이라 **실제로는 이 경로가 주로 돈다.**

**3순위 · 패턴 이름으로 찾기**
둘 다 없으면 패턴 이름만으로. 아는 3종일 때만 가능.

셋 다 안 되면 빈 목록.

> 아는 패턴이면 어느 경로로 들어왔든 **"이 패턴이면 흔한 원인"도 함께 합친다.** 모양으로 들어왔다고 잘 알려진 원인을 놓치면 안 되기 때문.

### 【3】 중복 정리

같은 `(공정, 고장모드, 원인, 검증신호)` 조합에 여러 경로가 닿으면 하나만 남긴다. 이때 **모양 정보가 있는 쪽을 남긴다** — 반대로 고르면 【4】의 필터가 쓸 신호가 사라져 무력화된다.

### 【4】 모양으로 걸러내기

같은 "가장자리 링"이라도 원 전체를 도는 링과 한쪽만 있는 호는 **원인이 다르다**. 전체면 계통적인 반경 문제, 한쪽이면 비대칭 핸들링이나 특정 장비 문제다.

| 어긋난 항목 | 감점 | 결과 |
|---|---|---|
| `angular_coverage` (전체 ↔ 부분) | −10 | **목록에서 제거** |
| 시계 방향이 완전히 다름 | −3 | 순위만 내림 |
| 진하기가 다름 | −1 | 순위만 내림 |
| 이어짐/끊김이 다름 | −1 | 순위만 내림 |

정렬은 **안정 정렬**이라 동점(0점 포함)은 원래 순서를 그대로 유지한다. 라이브 경로는 여기까지 끝난 뒤 `rank`를 1..n으로 다시 매긴다.

## 4. 설계에서 중요하게 고려한 것

### 4-1. 후보가 없을 때 예외 대신 "빈 목록"을 낸다

- **문제**: WM-811K에는 우리가 다루는 3종 말고도 여러 패턴이 있고 전부 `Unknown`이 된다. 예외를 던지면 배치 전체가 죽는다.
- **선택**: 빈 목록을 내보내고, **그래프의 조건부 엣지**가 이를 감지해 LLM 없는 응답 노드(⑦')로 보낸다.
- **왜 중요한가**: 후보가 0건일 때 **LLM 응답 노드로 가는 길 자체가 그래프에 존재하지 않는다**(#32). 나중에 ⑦에 진짜 LLM을 붙여도 재료 없이 문장을 지어낼 경로가 물리적으로 없다 — 환각 억제를 코드의 `if`가 아니라 **구조로** 보장하는 지점.
- **되돌릴 조건**: 미지 패턴에도 후보를 주고 싶다면 → 라이브 경로가 형상 진입으로 이미 부분 해결.

### 4-2. 자연어 매칭에 하한선(0.4)을 둔 것

- **문제**: 상위 k개를 뽑는 검색은 "안 닮아도 무조건 k개"를 돌려준다. 하한선이 없으면 아무 관계 없는 관측에도 그럴듯한 후보가 딸려 나온다 — 정확히 환각 억제 원칙 위배.
- **선택**: 하한 미달이면 매칭 0건. 아는 패턴이면 패턴 레벨 원인만, 모르는 패턴이면 빈 목록.
- **근거 (07-23 실측, `text-embedding-3-small`)**: 정답 매칭 **0.53~0.71** vs 오답 중 최고 **0.44~0.48** → 그 사이를 가르는 값으로 0.4 채택.
- **되돌릴 조건**: ⚠️ **임베딩 모델을 바꾸면 이 숫자는 무효다.** 유사도 계산 기준 자체가 달라져 재측정이 필요하다. 인덱스 빌드와 런타임이 같은 모델 상수를 공유하도록 짠 것도 같은 이유.

### 4-3. 모양 비교를 감점 전용으로 한 것

- **문제**: 모양이 맞는다고 가점을 주면, 문헌 근거가 약한 후보가 "모양만 맞아서" 위로 올라온다. KG가 문헌 수를 세어 매긴 순위를 모양 추측이 덮어쓰는 셈.
- **선택**: 맞으면 0점(그대로), 어긋나면 감점만. 안정 정렬이라 근거 순위가 보존된다.
- **대안과 기각**: 대칭 가감점(±)은 위 이유로 기각. 이 원칙이 깨지지 않도록 테스트로 못 박아뒀다(`test_demote_only_never_promotes_over_evidence`).

### 4-4. `angular`만 제거하고 나머지는 감점만

- **문제**: 모양이 근본적으로 안 맞는 후보를 순위만 낮춰 남기면 ⑤가 쓸데없이 MCP를 호출한다.
- **선택**: `angular` −10, 약한 신호는 다 합쳐도 −5. 제거 기준을 −10에 두면 **`angular`가 어긋났을 때만** 제거되도록 간격이 벌어진다. 수치 설계가 곧 규칙이다.
- **왜 환각 억제와 안 부딪히나**: "그럴듯함"으로 거르는 게 아니라 이미지 계산이 낸 **관측 사실**로 거르기 때문.
- **되돌릴 조건**: 제거가 정답을 죽인 사례가 나오면 `drop_contradictions=False`로 감점만 하게 되돌릴 수 있게 플래그를 남겨뒀다.

### 4-5. 라이브 조회가 KG 빌드 스크립트를 그대로 재사용

- **문제**: Neo4j 순회 로직을 backend에 복사하면 `kg_rca/6_ask_graphrag.py`와 두 벌이 되어 시간이 지나면 갈라진다.
- **선택**: 그 파일을 직접 불러다 쓴다(숫자로 시작하는 모듈명이라 `importlib`로 지연 로드, 1회 캐시).
- **대가**: 하나의 진실을 얻는 대신, kg_rca 쪽 함수 이름·인자가 바뀌면 backend가 **조용히** 깨진다(§8-5).

| **의미 진입** | 자연어 문장을 숫자 벡터로 바꿔 가장 비슷한 형상을 찾는 방식 |

| **tier(검증등급)** | 그 원인을 어떻게 확인할 수 있는지의 분류 (자동/반자동/근거없음) |
| **후보(candidate)** | 아직 확인되지 않은 원인 가설 1건 |
