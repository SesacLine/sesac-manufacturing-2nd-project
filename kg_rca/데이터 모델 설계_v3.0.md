# 데이터 모델 설계_v3.0

> 갱신: 2026-07-22 · v2.0(07-13)을 **공간 진입점(SpatialSignature) + 모폴로지(설계 B)** 도입에
> 맞춰 개정. v2.0과 달라진 부분은 각 절에 **[v2.0 대비]** 로 표시.
> KG 상세 정본: `../docs/KG_schema_v1.2.md` / 출력 계약: `KG_output_명세.md` /
> VLM→KG 입력 계약: `VLM_output - KG 연결.md` / 정합성: `MCP_KG_정합성검토.md`
> 이번 개정의 핵심 소스: 목업 `data/docs/doc_A~H`, 재랭킹 `backend/graph_client/morphology_rank.py`.

이번 버전의 한 줄 요약: **관측(VLM+die-matrix)의 "형상"이 KG의 두 번째 진입점이 됐고, "모폴로지"
(density/continuity/angular/clock)는 노드가 아니라 `FORMS_IN` 엣지 속성 + 소프트 랭킹 신호로
들어온다(설계 B).** shape/zone enum 자체는 v2.0 그대로 유지한다(재토큰화 안 함).

데이터 저장소는 3종류다 — ① fab.db(SQLite) ② Neo4j(그래프 DB) ③ app_state.db(SQLite) +
LangGraph `RCAState`. **KG 출력 파일(`outputs/hypotheses.json`)이 Neo4j와 LangGraph 사이의
인터페이스**다.

```
fab.db ──(시뮬레이터가 빌드)──> MCP 서버(T1~T9) ──> LangGraph 서비스
                                                        ▲
문헌(data/raw/ + data/docs/) ──(KG 파이프라인 0~5번)──> Neo4j ──(6번)──> hypotheses.json
   └ [v2.0 대비] data/docs 추가                                       (③ 노드가 읽는 계약)
```

## 0. KG 입력 계약 (진입점) — **[v2.0 대비 신설]**

v2.0까지 KG 조회의 진입점은 `DefectPattern` 하나였다. v3.0에서 **관측의 형상 시그니처가 두 번째
하드 진입점**이 되고, 모폴로지가 소프트 신호로 추가된다. 관측 payload 정본은 `VLM_output - KG 연결.md`.

### 진입점은 2개 (하드 매칭 키)

| # | 입력 키 | 고정 vocabulary | 생산자 |
| --- | --- | --- | --- |
| ① | `DefectPattern` | `Center` \| `Scratch` \| `Edge-Ring` | CNN(클래스) |
| ② | `SpatialSignature` = `{shape}@{zone}` | shape `ring\|cluster\|line\|blob\|global\|random` × zone `center\|mid\|edge\|any` | die-matrix 통계 + VLM 서술 |

- 두 진입점이 성립하는 이유는 **문헌과 관측이 둘 다 같은 어휘를 말하기 때문**이다(공유 조인 어휘).
  ①은 `ARISES_IN`/`HAS_SIGNATURE`, ②는 `FORMS_IN`이 문헌 쪽 공급이다(§2 참조).
- ②의 실전 용도는 **미지 패턴(CNN=`Unknown`)**: 클래스 이름이 없어도 `shape@zone`으로 순회 진입.
  목업 `doc_H`가 이 `FORMS_IN`(형상→공정)을 공급한다.

### 모폴로지 (소프트 신호 · `FORMS_IN` 엣지 속성 · 설계 B)

| 축 | 값 | 역할 | 근거 |
| --- | --- | --- | --- |
| `angular_coverage` | `full \| partial \| unknown` | **판별자(강)** | die-matrix에서 결정적 산출, full↔partial이 인과를 가름 |
| `density` | `high \| medium \| low \| unknown` | 소프트 랭킹 | VLM 텍스트 기반, 노이즈 |
| `continuity` | `continuous \| intermittent \| discontinuous \| not_applicable \| unknown` | 소프트 랭킹 | 〃 |
| `clock_positions` | `[1..12]` (partial일 때만) | 소프트(angular partial 보조) | 〃 |

- **왜 노드가 아니라 엣지인가**: `SpatialSignature` 노드 id는 `shape@zone` 하나뿐이라, 모폴로지를
  노드 속성으로 넣으면 같은 `ring@edge`가 서로 다른 density로 MERGE되며 **덮어써진다(clobber)**.
  복합키(`shape@zone@density@…`)로 가면 **조합 폭증(≈1,440종) + 정확일치 brittle + unknown 죽은 노드**.
  그래서 모폴로지는 **`FORMS_IN` 관계 속성**으로 싣고, 매칭은 §3의 감점 전용 재랭킹으로 처리한다.

### 입력 예시 (복붙용)

```json
{
  "pattern": "Edge-Ring",
  "signature": { "shape": "ring", "zone": "edge" },
  "morphology": {
    "density": "high", "continuity": "continuous",
    "angular_coverage": "full", "clock_positions": []
  }
}
```
```json
{
  "pattern": "Unknown",
  "signature": { "shape": "ring", "zone": "edge" },
  "morphology": {
    "density": "low", "continuity": "discontinuous",
    "angular_coverage": "partial", "clock_positions": [5, 6, 7]
  }
}
```

- `pattern`이 3종 중 하나면 ①로, `Unknown`이면 `signature`로 ② 진입.
- `shape`/`zone`은 **반드시 일치**(하드키), `morphology` 4종은 **상충 시 감점만**(소프트).
- **경계 down-mapping**: 관측 payload(`VLM_output - KG 연결.md`)의 자유 어휘(shape의 `band`/`arc`,
  zone의 `interior`)는 KG enum으로 접어서 넣는다(`band/arc→ring`, `interior→mid`). KG enum
  재토큰화는 이번 범위 밖(파급이 큼 — §6 K-3).

## 1. fab.db (SQLite) *(v2.0 유지)*

`metric_series.scope`는 장비 단위다. 로트별 수율(저수율 선별)은 `wafer.die_map`을 직접 집계.
`fab.md`가 `metric_series`·`wafer`와 어긋나는 문제는 §6 F-2 그대로.

**[v3.0 관점 추가]** 진입점 ②의 `angular_coverage`/`clock_positions`는 fab.db가 아니라
**판독 시점의 die-matrix 통계**에서 결정적으로 계산된다(로트/웨이퍼 die 좌표 분포). fab telemetry와
무관하며, 이 값들의 provenance는 `spatial_source: die_matrix_statistics`(관측 payload).

## 2. Neo4j 그래프 스키마 — **[v2.0 대비: FORMS_IN 엣지 확장]**

노드 라벨은 v2.0과 동일하다(신규 노드 없음 — 설계 B 핵심). `SpatialSignature` id는 여전히
`{shape}@{zone}`.

### 노드 라벨 (v2.0 유지)

| 라벨 | 유일키 | 주요 속성 | 출처 |
| --- | --- | --- | --- |
| `DefectPattern` | `id`(3종) | `aliases`, `expected_zone`, `expected_shape` | 시드 |
| `ProcessStep` | `id`(6종) | `aliases` | 시드 (`lot_history.step`) |
| `Parameter` | `id`(21종) | `steps`, `aliases`, `fab_table` | 시드 (`telemetry.param`) |
| `SpatialSignature` | `id`=`{shape}@{zone}` | `shape`, `zone` (코드 enum) | 문헌 추출 |
| `FailureMode` | `id` | `name`, `description` | 문헌 추출 |
| `Cause` | `id` | `name`, `description`, `unverifiable_signals` | 문헌 추출 |
| `Maintenance` | `id` | `name`, `consumable`, `fab_table` | 문헌 추출 |
| `Recipe` | `id` | `name`, `fab_table` | 문헌 추출 |

### 관계 타입 — **[v2.0 대비: FORMS_IN에 모폴로지 4속성 추가]**

| 관계 | 방향 | 속성 |
| --- | --- | --- |
| `ARISES_IN` | `(DefectPattern)→(ProcessStep)` | `occurrence_prior`, `quotes`, `chunk_ids` |
| `HAS_SIGNATURE` | `(DefectPattern)→(SpatialSignature)` | `quotes`, `chunk_ids` |
| **`FORMS_IN`** | `(SpatialSignature)→(ProcessStep)` | `occurrence_prior`, **`density`, `continuity`, `angular_coverage`, `clock_positions`** (신규), `quotes`, `chunk_ids` |
| `OCCURS_IN` | `(FailureMode)→(ProcessStep)` | `quotes`, `chunk_ids` |
| `CAUSED_BY` | `(FailureMode)→(Cause)` | `quotes`, `chunk_ids` |
| `ATTRIBUTED_TO` | `(DefectPattern)→(Cause)` | `quotes`, `chunk_ids` |
| `VERIFIED_BY` | `(Cause)→(:Evidence)` | `target_label`, `direction` |

- **모폴로지 4속성은 `FORMS_IN` 전용**이다. LLM이 문헌의 형상 서술에서 함께 추출한다
  (`5_build_kg_from_chunks.py`의 `Relationship` 모델 + 프롬프트 FORMS_IN 섹션).
  적재는 `MERGE (g)-[rel:FORMS_IN]->(s)` + `SET rel.density = …` 등.
- ⚠ **엣지 clobber 주의**: `FORMS_IN`은 `(signature, step)`으로 MERGE라, 같은 `ring@edge→ETCH`를
  서로 다른 모폴로지로 두 번 서술하면 나중 값이 앞을 덮는다. 목업은 일관 작성으로 회피, 실문헌
  확장 시 유의(누적이 필요하면 `chunk_ids`처럼 리스트 병합 방식으로 바꿔야 함).

### 문헌 소스와 진입점 매핑 — **[v2.0 대비: data/docs 추가]**

`2_load_txt.py`가 이제 **`data/raw` + `data/docs` 두 폴더**를 읽는다(하위 디렉토리 제외 유지).

| 소스 | 성격 | 공급하는 진입점/층 |
| --- | --- | --- |
| `data/raw/*` (7) | 실문헌(교과서·논문 발췌) | ①(ARISES_IN/ATTRIBUTED_TO), ②(일부 FORMS_IN — Liao 논문) |
| `data/docs/doc_A` | 전문가 암묵지 목업 (패턴→공정) | ① ARISES_IN + `HAS_SIGNATURE`(②노드 정의) |
| `data/docs/doc_B~G` | 공정별 트러블슈팅 목업 | 내부 체인 `OCCURS_IN`/`CAUSED_BY`/`VERIFIED_BY` (ProcessStep에서 조인) |
| `data/docs/doc_H` | **형상·모폴로지 암묵지 목업 (신규)** | ② `FORMS_IN`(형상→공정) + 모폴로지 4속성 |

- 목업(`data/docs`)은 **원래 파이프라인에 미연결**이었다가 v3.0에서 로더에 편입됐다.
  `doc_H`는 클래스명을 일부러 안 써서 `FORMS_IN`(형상 진입)으로 추출되도록 설계됐다.

### 시딩과 갱신 *(v2.0 유지)* / 런타임 조회 *(v2.0 유지 + 출력 확장)*

파이프라인 `0_reset → 2_load → 3_split → 4_ingest(시드 3앵커) → 5_build(LLM 추출) → 6_ask`.
추출 비결정성 때문에 서비스는 그래프가 아니라 `hypotheses.json` 스냅샷을 `meta.generated_at`으로
신선도 검사하며 읽는다.

가설 1건의 핵심 키 (전체는 `KG_output_명세.md`) — **[v2.0 대비: path.morphology 추가]**:

```
tier            자동|반자동|근거없음
scenario_hint   A3|A5|A2|A6|null
path            {signature, morphology, step, failure_mode, cause, evidence, evidence_label}
                 └ morphology = {density, continuity, angular_coverage, clock_positions} | null
                   (형상 경유 FORMS_IN일 때만 채워짐. step/direct 경로는 null)
verification    {fab_table, direction, unverifiable_signals}
score           {occurrence_prior, evidence_docs, evidence_chunks}
provenance      {chunk_ids, quotes}
mapping         {matched_cause, prob, citation, ...} | null
```

순위(빌드 시점) = `(occurrence_prior, evidence_docs, evidence_chunks)` 내림차순 — v2.0 그대로.
**모폴로지 재랭킹은 빌드가 아니라 런타임(§3)에서** 관측과 비교해 적용한다(빌드 시엔 관측이 없음).

## 3. LangGraph `RCAState` — **[v2.0 대비: 모폴로지 재랭킹 도입]**

### GraphRAGCandidate 필드 추가

| 필드 | v2.0 | **v3.0 (변경)** |
| --- | --- | --- |
| `signature` | `{shape}@{zone}` \| null | 유지 |
| `morphology` | (없음) | **신규.** `hypotheses.json`의 `path.morphology`를 그대로 옮김. 형상 경유가 아니면 null |

### ③ 조회 노드: angular 판별자 재랭킹 — **[v2.0 대비 신설]**

- `KGClient.get_candidates(pattern, observation=None)` — 관측 모폴로지를 주면
  **`morphology_rank.rerank_by_observation`** 으로 후보를 재정렬한다.
- 방침 = **감점 전용(demote-only)**:
  - `angular_coverage` full↔partial 상충 → **−10(강등)**, partial 시계 완전 불일치 → −3
  - `density`/`continuity` 상충 → 각 −1(소프트)
  - 일치 / unknown / not_applicable / 무관측 → 0(중립) → **kg_rca 순위 인위적 상승 없음**
- 안정 정렬이라 동점(0점)은 근거 기반 원래 순위를 보존한다. 상충 후보만 아래로 가라앉는다.
- **판정 근거**: shape@zone은 이미 하드 매칭(후보 존재 자체가 매칭 성공)이므로 재랭킹에선 안 본다.
  angular만 강하게, 나머지는 약하게 — VLM 노이즈 반영.

### ⚠ 아직 배선 안 된 부분 (관측층)

- `group["observation"]`에 관측 모폴로지를 싣는 **VLM/die-matrix 관측층이 미구축**이다
  (`vlm.py`가 `pattern="Center"` 하드코딩, 모폴로지 미산출). 그래서 현재 `observation=None`으로
  전달되고 재랭킹은 **통과(no-op)** — kg_rca 순위 그대로다.
- 판별자 로직·배선·단위 테스트(`backend/tests/test_morphology_rank.py`)는 **완료**되어, 관측층이
  `group["observation"]`을 채우는 순간 자동 작동한다.

§3.1의 evidence 구조·스코어링·Critic 규칙, `prob` null 처리, `drift_detected` 조건은 v2.0 그대로.
